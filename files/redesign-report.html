Part One

Existing Site Audit

The current site is a sincere, honest first effort. It works. Navigation is clear, dark mode is implemented, content is organised across four meaningful sections. But it is also, at every level, the default template — and for a site that aims to communicate intellectual ambition, that gap matters.

Structural Strengths to Preserve

  • Four-section architecture (Home / Projects / Mindspace / Codespace) is exactly right. It is a genuine conceptual structure, not just tab labels.
  • Dark mode persistence via localStorage — works correctly, uses system preference as fallback.
  • Flat file structure with semantic folder naming (/projects/astrostats/, /projects/volatility/) is sensible and scalable.
  • Real academic content already exists: a published paper (GP Kepler Imputation), substantive essay writing (Random Forests/SLLN analogy), and genuine research (Penn State AstroStats, Rough Volatility).
  • The writing voice is good — thoughtful, confident, not overclaiming.

Design Weaknesses

Critical

Header color: #0ffaa0. This electric neon-green is the single biggest design problem. It communicates "coding tutorial blog," not research portfolio. White text on it also fails contrast.

Critical

Page title: "Homepage of Shyam's blogpost." Undersells the site drastically. A PhD committee member sees this and clicks away.

Major

Zero visual hierarchy. Every page is: neon header → gray nav → white box → footer. No section has its own visual personality. Projects and Mindspace look identical.

Major

CSS is copy-pasted across all 7+ HTML files. Changing the header color requires editing every file. No shared stylesheet exists.

Major

Mindspace and Codespace are placeholder pages. Both link to the AstroStats project as their only content. The sections have no distinct identity or content.

Major

Volatility page has copy-paste errors — the header reads "AstroStats Summer Program" and highlights are about Bayesian astrophysics, not volatility models.

Moderate

No responsive layout. Everything is a single block with max-width. No grid, no two-column layout, no responsive navigation.

Moderate

No typography scale. Only Lato, rendered at browser defaults. No size/weight system, no display font for headers, no monospace for code contexts.

Moderate

Projects page is a plain <ul>. Research contributions are listed like grocery items. No context, no dates, no domain tags.

Minor

Broken relative links. The random forests page links back to ..projects/astrostats_summer_2025.html (missing a slash). The nav in astrostats_summer_2025.html links to ../projects/projects_index.html which is wrong from its depth.

Content Audit Summary

The site has more real content than it appears to — three genuine research projects, one published paper, one substantive essay, and two GitHub repos with actual code. The design currently hides this depth. A visitor looking for research work has to dig through a neon header and grocery-list formatting to find a first paper. That's backwards.

Key Insight

The content is good. The gap is entirely presentation. This is the ideal redesign scenario: no content strategy overhaul needed, only visual and structural work.

Part Two

Design Philosophy

"A young researcher's digital observatory — where curiosity, rigor, and intellectual depth are the design materials."

The Observatory Metaphor

An observatory is a place of patient, careful observation. It is not a lab (too clinical), not a journal (too academic), and not a blog (too casual). It is a workspace that is also a sanctuary — serious but personal, rigorous but exploratory. The design should feel like entering a well-organized study with a large window facing a dark sky.

This translates into: deep ink tones, parchment surfaces, considered typography, structural clarity, and zero decoration for decoration's sake.

The Four Section Personalities

Rather than identical pages with different content, each section should feel like a distinct room in the same building. Same architectural language, different atmosphere.

⌂
Home
Editorial. The foyer. Stakes a claim about who Shyam is, what he works on, and where he is going. Authoritative but warm.
◎
Projects
Archive-like. Dense with information. Research cards with tags, abstracts, dates, and links. Academic paper aesthetic.
∿
Mindspace
Journal-like. Essays, reflections, mathematical intuitions. Long-form reading experience. Wide margins, generous line spacing.
</>
Codespace
Laboratory notebook. Technical, but with narrative. Monospace accents, structured notes, links to notebooks and repos.

What to Avoid (and Why)

  • Startup aesthetics (gradient hero sections, "Get started" CTAs, feature grids) — communicate a product, not a person with a research identity.
  • Dark hacker themes (terminal greens, all-black, code rain) — conflate statistical research with software engineering culture.
  • Excessive animation — a PhD committee member who arrives via a Google Scholar search link should be able to find the paper in 10 seconds. Animation that delays this is hostile.
  • Social-media-style layouts (cards with engagement metrics, "follow me" prompts, algorithmic content ordering) — wrong register entirely for research identity.

Part Five

Homepage Redesign

Design Goals

The homepage has one job: within 5 seconds, a visiting researcher should understand who Shyam is, what he works on, and how seriously to take this site. Right now it reads like a first draft ("Welcome, Shyam here!"). It should read like a scientific instrument — precise, intentional, quietly confident.

Layout Concept: The Observatory Entry

A dark hero section occupying full viewport width (not height) with a grid overlay, followed by a light content section. The hero establishes identity; the content section gives depth.

B.Stat (Hons.) · Indian Statistical Institute, Kolkata
Statistics as a
lens on the universe
Research at the intersection of astrostatistics, Bayesian inference, Gaussian processes, and quantitative finance.
Astrostatistics Bayesian Inference Gaussian Processes Rough Volatility

Content Below the Hero

Three columns on desktop, stacked on mobile:

  • Recent Work — last 2–3 projects with tags and one-line descriptions.
  • About — 3-4 sentences. Institution, interests, current focus. No lists.
  • Contact / Links — email, LinkedIn, GitHub, resume. Clean, no icon clutter.

Things to Remove from Current Homepage

  • The phrase "Fantasy and Realism combined" — charming but meaningless to a researcher.
  • "Welcome, Shyam here!" — too informal for the intended audience.
  • The <strong>Name:</strong> / <strong>Program:</strong> label-value list format. Use prose instead.
  • Having two resume PDFs with different names. Pick one.

Shared JavaScript: theme.js

Replace the inline <script> repeated in every file with a single shared file. Also adds a keyboard shortcut and properly reads system preference.

// assets/js/theme.js
(function() {
  const stored = localStorage.getItem('theme');
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  if (stored === 'dark' || (!stored && prefersDark)) {
    document.documentElement.classList.add('dark');
  }
})();

function toggleTheme() {
  const isDark = document.documentElement.classList.toggle('dark');
  localStorage.setItem('theme', isDark ? 'dark' : 'light');
}

// Keyboard shortcut: Alt+D toggles theme
document.addEventListener('keydown', e => {
  if (e.altKey && e.key === 'd') toggleTheme();
});

Note: Apply dark class to <html> not <body>, and do it in a <script> in the <head> to avoid a flash of unstyled content (FOUC) on page load.

Responsive Strategy

Three breakpoints, mobile-first:

/* Mobile first — single column, 0–599px */
.home-grid { display: grid; gap: var(--space-6); }

/* Tablet — 600px–899px */
@media (min-width: 600px) {
  .home-grid { grid-template-columns: 1fr 1fr; }
}

/* Desktop — 900px+ */
@media (min-width: 900px) {
  .home-grid { grid-template-columns: 2fr 1fr 1fr; }
  .site-header { display: flex; align-items: center; }
}
